01
Fibonacci Cine & Edición

Montaje Cinematográfico

El problema real

En el cine de suspenso o acción, mantener planos de la misma duración aburre al espectador. Los planos largos generan calma; los cortos, urgencia. Pero cambiar duraciones sin lógica matemática produce caos visual que desgasta sin emocionar.

El editor busca crear una aceleración orgánica de la tensión — un clímax que se construya solo, sin que el espectador lo note conscientemente. Para eso necesita un patrón de duraciones que crezca de forma natural, no arbitraria.

"El montaje no es cortar planos — es orquestar el tiempo. La duración de cada plano dicta cómo late el corazón del espectador."

— Andréi Tarkovski, director soviético (1932–1986)

¿Por qué Fibonacci resuelve esto?

La serie 1, 1, 2, 3, 5, 8, 13, 21... crece de forma orgánica: cada número es la suma de los dos anteriores. Al usarla en orden descendente — de los planos más largos a los más cortos — se produce un aumento de adrenalina que imita la aceleración natural de los latidos del corazón. El espectador la siente en el cuerpo antes de entenderla con la mente.

Cómo funciona el algoritmo

Se suman términos de Fibonacci hasta llegar lo más cerca posible de N segundos. Luego se muestran esos términos en orden descendente: el primer plano es el más largo (calma), el último dura 1 segundo (impacto).

// Avanzar sumando Fibonacci hasta acercarse a N
let suma = 0;
let i = 1;

while (suma < N) {
  let fib = fibonacci(i);
  if (suma + fib > N) break;
  suma += fib;
  i++;
}

// Mostrar en orden descendente
for (let j = i - 1; j >= 1; j--) {
  let fib = fibonacci(j);
  resultado += fib + " segundos → ";
}

¿Cómo ayuda Fibonacci a este problema?

Sin un patrón matemático, el editor decide las duraciones "a ojo" — y el resultado es arbitrario. Con Fibonacci, la aceleración del montaje sigue el mismo ritmo de crecimiento que usa la naturaleza: cada plano es la suma de los dos anteriores, igual que los pétalos de un girasol o las ramas de un árbol. El espectador no ve la matemática, pero la siente: su ritmo cardíaco sube junto a los planos porque su cerebro reconoce ese patrón como algo orgánico y real. Eso es exactamente lo que busca un buen editor.

02
Números Primos Videojuegos · Game Design

Spawn de Enemigos

El problema real

En los juegos de naves (shoot 'em ups) o de supervivencia como Vampire Survivors, si los enemigos aparecen en oleadas fijas cada 5 o 10 segundos, el jugador aprende el patrón de memoria muy rápido y el juego se vuelve aburrido. Los desarrolladores necesitan que los enemigos aparezcan con un ritmo que se sienta caótico e impredecible, pero que en realidad esté controlado por código.

Si los enemigos simplemente aparecieran en momentos aleatorios sin ninguna lógica, el juego sería injusto — podrían aparecer cinco seguidos en un segundo y luego nada durante veinte. Necesitás algo que sea asimétrico pero estable.

"El buen diseño de juegos es hacer que el jugador sienta que está en control, mientras en realidad vos controlás cada segundo de su experiencia."

— Shigeru Miyamoto, creador de Mario, Zelda y Donkey Kong

¿Por qué los primos resuelven esto?

Los números primos (2, 3, 5, 7, 11, 13, 17, 19, 23...) no tienen divisores comunes entre sí. Eso significa que sus apariciones casi nunca coinciden de forma regular. Si configurás distintos tipos de enemigos para que aparezcan en segundos primos, el jugador percibirá un flujo constante, asimétrico y desafiante — sin que el juego sea injusto. La dificultad está controlada matemáticamente, no al azar.

Cómo funciona el algoritmo

El bucle simula el paso del tiempo segundo a segundo. En cada segundo se verifica si es primo con esPrimo(). Si lo es, aparece un enemigo. El usuario elige cuántos segundos simular.

// Simular el paso del tiempo en el juego
for (let segundo = 1; segundo <= duracion; segundo++) {

  if (esPrimo(segundo)) {
    // Segundo primo: aparece enemigo
    resultado += "Seg. " + segundo + ": ENEMIGO";
  } else {
    // Segundo normal: calma para el jugador
    resultado += "Seg. " + segundo + ": —";
  }
}

Simulá una partida

¿Cómo ayudan los primos a este problema?

Una oleada fija cada 5 segundos se aprende en dos intentos. Una aparición en cada segundo primo es impredecible para el jugador pero completamente determinista para el código: siempre va a ser los mismos segundos, el juego es justo. Los primos distribuyen los eventos de forma asimétrica sin necesitar aleatoriedad real, lo que hace el diseño reproducible, testeable y balanceable. Eso es exactamente lo que necesita un desarrollador de videojuegos.

03
Fibonacci + Primos Pixel Art · Arte Digital

Texturizado Orgánico

El problema real

En el arte digital y en juegos de construcción de bloques, si pintás una pared grande usando un solo color plano o un patrón que se repite regularmente, el resultado se ve artificial y aburrido. Los artistas usan "texturizado aleatorio controlado" para romper la monotonía y simular materiales reales como piedra, madera o metal desgastado.

El reto: hacer que la computadora decida automáticamente dónde poner bloques con texturas especiales (relieve, brillo, desgaste), logrando que el resultado se vea disperso y natural, sin ser verdaderamente aleatorio — porque si fuera aleatorio no se podría reproducir ni controlar.

"La naturaleza no repite sus patrones de forma exacta, pero tampoco los genera al azar. Usa reglas matemáticas simples que producen complejidad visual."

— Benoît Mandelbrot, matemático, padre de la geometría fractal

¿Por qué Fibonacci + Primos?

Fibonacci define el ancho de cada sección del lienzo — sus proporciones son armónicas y el ojo las percibe como naturales (igual que una concha de caracol o una flor). Los primos dentro de esa serie actúan como filtro de asimetría: si el ancho de esa sección es un número primo, el algoritmo coloca un bloque con textura especial. Como los valores primo de Fibonacci aparecen de forma esporádica e impredecible, el patrón visual resultante imita perfectamente el desgaste natural de la piedra.

Cómo funciona el algoritmo

Para cada sección del lienzo se calcula su ancho con fibonacci(i). Si ese ancho es primo, la sección recibe un bloque texturizado especial. Si no, recibe un bloque liso.

for (let i = 1; i <= secciones; i++) {

  let anchoBloque = fibonacci(i);

  if (esPrimo(anchoBloque)) {
    // Bloque con textura especial / relieve
    resultado += "Sección " + i +
      " (ancho: " + anchoBloque + "): ESPECIAL";
  } else {
    // Bloque liso base
    resultado += "Sección " + i +
      " (ancho: " + anchoBloque + "): liso";
  }
}

¿Cómo ayudan Fibonacci y los primos a este problema?

Usar solo colores aleatorios produce caos que no se puede controlar ni reproducir. Usar un patrón fijo produce algo aburrido y artificial. Fibonacci + primos resuelven ambos problemas: Fibonacci da las proporciones armónicas que hacen que las secciones se vean bien juntas, y los primos dentro de esa serie determinan de forma asimétrica y no periódica dónde va la textura especial. El resultado parece hecho a mano por un artista, pero fue generado por diez líneas de código.
